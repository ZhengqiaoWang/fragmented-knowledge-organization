# 知识碎片整理插件 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
一款 Obsidian 插件，帮助用户快速记录知识碎片，并利用 AI 能力对碎片进行批量整理，形成结构化的知识体系。

### 1.2 核心价值
- **快速捕获**：降低记录门槛，随时记录灵感、链接、图片等碎片信息
- **智能整理**：利用 AI 批量处理碎片，发现关联、去重合并、分类归纳
- **知识沉淀**：将零散碎片转化为结构化笔记，形成个人知识库
- **独立管理**：每个碎片独立存储，便于引用、链接和单独处理

### 1.3 目标用户
- 知识工作者、研究者、学习者
- 使用 Obsidian 进行知识管理的用户
- 有大量碎片信息需要整理的用户

## 2. 功能需求

### 2.1 快速记录模块

#### 2.1.1 记录入口

**侧边栏常驻面板（推荐）**
- 打开方式：点击侧边栏图标或命令 "打开知识碎片侧边栏"
- 类型选择器：📝 文本 | 🔗 链接 | 🖼️ 图片
- 对应输入框根据类型动态切换

**命令面板快捷入口**
- `Ctrl+P` → "快速记录文本" → 弹出多行输入框
- `Ctrl+P` → "快速记录链接" → 输入 URL + 可选备注
- `Ctrl+P` → "快速记录图片" → 选择本地图片文件 + 可选备注

#### 2.1.2 侧边栏界面

```
┌─────────────────────────────────┐
│  📝 知识碎片                     │
├─────────────────────────────────┤
│  [📝 文本] [🔗 链接] [🖼️ 图片]   │  ← 类型选择器
│  ┌───────────────────────────┐  │
│  │ 输入内容...               │  │  ← 动态切换的输入框
│  └───────────────────────────┘  │
│  [发送]                         │
│                                 │
│  📅 今日碎片 (3)        [AI 整理] │
│  ┌───────────────────────────┐  │
│  │ ☐ 14:30 文本内容...       │  │
│  │ ☐ 14:25 🔗 链接预览...    │  │
│  │ ☐ 14:20 🖼️ 图片预览...    │  │
│  └───────────────────────────┘  │
│                                 │
│  ◀ 2024-01-14 | 2024-01-16 ▶   │  ← 日期切换
└─────────────────────────────────┘
```

#### 2.1.3 支持的碎片类型

| 类型 | 输入方式 | 存储格式 |
|------|---------|---------|
| **文本** | 直接输入 | Markdown 正文 |
| **链接** | URL 输入 + 备注 | Markdown 链接 + 备注 |
| **图片** | 选择文件 + 备注 | 图片引用 + 附件存储 |

#### 2.1.4 图片上传功能

**上传流程：**
1. 点击"🖼️ 图片"切换类型
2. 点击"📁 选择图片"按钮
3. 系统文件选择器选择图片
4. 自动上传到当天日期目录下的 `附件/` 文件夹
5. 显示图片预览和文件名
6. 可选：输入备注
7. 点击"发送"保存

**图片管理：**
- 自动重命名：`YYYYMMDDHHMMSS-原文件名`
- 预览功能：上传后显示缩略图
- 删除功能：点击 ✕ 取消选择
- 存储位置：`知识碎片/2026-04-17/附件/`

#### 2.1.5 交互细节
- 文本输入支持多行
- 链接输入自动验证 URL 格式（必须以 http:// 或 https:// 开头）
- 所有记录可添加可选备注
- 移动端适配：使用系统文件选择器

### 2.2 碎片存储模块

#### 2.2.1 新的存储结构（每个碎片一个文件）

```
用户库根目录/
└── 知识碎片/              # 可自定义路径
    └── 2026-04-17/        # 日期文件夹
        ├── 附件/          # 当天附件
        │   ├── 20260417085632-image.jpg
        │   └── 20260417091234-screenshot.png
        ├── 2026-04-17 知识碎片.md  # 索引文件
        ├── 10-30-文本.md           # 独立碎片文件
        ├── 14-25-链接.md
        └── 16-45-图片.md
```

#### 2.2.2 碎片文件格式（带 frontmatter 元数据）

```markdown
---
type: text
timestamp: 2026-04-17T10:30:00.000Z
---

# 10:30 文本

这是碎片内容...
```

**链接类型：**
```markdown
---
type: link
timestamp: 2026-04-17T14:25:00.000Z
url: https://example.com
remark: 可选备注
---

# 14:25 链接

https://example.com

- 备注：可选备注
```

**图片类型：**
```markdown
---
type: image
timestamp: 2026-04-17T16:45:00.000Z
imagePath: 知识碎片/2026-04-17/附件/20260417164500-image.jpg
remark: 可选备注
---

# 16:45 图片

![](知识碎片/2026-04-17/附件/20260417164500-image.jpg)

- 备注：可选备注
```

#### 2.2.3 索引文件格式

```markdown
# 2026-04-17 星期四

## 碎片列表

- [[10-30-文本.md|10:30 📝]]
- [[14-25-链接.md|14:25 🔗]]
- [[16-45-图片.md|16:45 🖼️]]
```

#### 2.2.4 文件命名规则
- 碎片文件：`HH-MM-类型.md`（例如：`10-30-文本.md`）
- 索引文件：`YYYY-MM-DD 知识碎片.md`
- 图片文件：`YYYYMMDDHHMMSS-原文件名`

#### 2.2.5 优势说明

**相比旧方案（一个文件放所有碎片）：**
- ✅ 每个碎片独立，便于单独引用和链接
- ✅ 可添加独立 frontmatter 元数据（标签、分类、状态）
- ✅ 更适合 Obsidian 的双向链接特性
- ✅ AI 整理后可以直接移动文件
- ✅ 支持 Dataview 等插件查询
- ✅ 版本控制更精细

### 2.3 AI 整理模块

#### 2.3.1 整理入口
- 在侧边栏选择多个碎片（点击复选框）
- 点击"AI 整理"按钮
- 弹出整理对话框

#### 2.3.2 整理流程

```
选择碎片（多选）
    │
    ↓
点击"AI 整理"按钮
    │
    ↓
打开 OrganizeModal
    │
    ├─ 显示已选碎片列表
    ├─ 选择整理方式
    └─ 点击"开始整理"
        │
        ↓
  AIOrganizerService
        │
        ├─ 步骤 1: AI 分类
        ├─ 步骤 2: 生成知识卡片
        │   ├─ 提取标签
        │   ├─ 复制图片到知识体系
        │   └─ 保存文件
        └─ 步骤 3: 发现关联（可选）
            │
            ↓
      创建知识卡片文件
```

#### 2.3.3 整理方式

| 方式 | 功能描述 | 输出结果 |
|------|---------|---------|
| 📝 分类归纳 | 自动归类到不同主题 | 按主题分类的知识卡片 |
| 🔄 去重合并 | 识别相似内容并合并 | 合并后的去重列表 |
| 🔗 发现关联 | 找出碎片间的联系 | 带关联链接的知识卡片 |
| 📚 生成知识卡片 | 整理成结构化笔记 | 标准化的知识卡片 |

#### 2.3.4 AI 提供商支持

**云端 AI（OpenAI 兼容）**
- 配置项：API Key、模型、基础 URL
- 支持：OpenAI、Azure OpenAI、其他兼容 API

**本地 AI（Ollama）**
- 配置项：服务器地址、模型
- 支持：Ollama、其他本地部署模型

#### 2.3.5 图片处理
AI 整理时自动将图片从 `知识碎片/日期/附件/` 复制到 `知识体系/分类/附件/`，并更新图片路径为相对路径。

## 3. 非功能需求

### 3.1 性能要求
- 插件启动时间 < 1 秒
- 记录碎片响应时间 < 500ms
- AI 整理加载时间取决于网络，需显示加载状态

### 3.2 兼容性要求
- Obsidian 版本 >= 1.4.0
- 支持 Windows、macOS、Linux、iOS、Android
- 支持亮色/暗色主题

### 3.3 数据安全
- API Key 加密存储
- 本地 AI 优先（用户可选择）
- 不上传用户数据到第三方（除用户主动配置的 AI API）

### 3.4 用户体验
- 界面简洁，符合 Obsidian 原生风格
- 操作流畅，支持快捷键
- 错误提示清晰友好
- 移动端适配

### 4.3 核心数据模型

```typescript
interface Fragment {
  id: string;                    // UUID
  type: 'text' | 'link' | 'image';
  content: string;
  metadata: {
    url?: string;                // 链接类型
    imagePath?: string;          // 图片类型
    remark?: string;             // 备注
  };
  timestamp: number;             // 时间戳
  status: 'pending' | 'processing' | 'processed';
  tags?: string[];               // 标签
}

interface PluginSettings {
  fragmentFolder: string;         // 碎片文件夹路径
  attachmentFolder: string;       // 附件文件夹名称
  aiProvider: 'openai' | 'ollama' | 'custom';
  apiKey: string;
  model: string;
  baseUrl?: string;               // 自定义 AI 地址
  autoCreateAttachments: boolean;
  autoRenameImages: boolean;
  autoFocusInput: boolean;
  knowledgeBaseFolder: string;    // 知识卡片输出目录
}
```
